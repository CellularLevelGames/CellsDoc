void AnimateMeleeAttack ( )

Executed on each client.

For UX, the hit is also registered client-side, i.e. dodging will be correct at the evading client. For this, the animation calls a callback upon the instance of the hit. The attack speed will change the animation itself, also leading to an earlier event.

void HitMeleeAttack ( )
inlineprivate

Called from the melee attack animation.

This is the instance of damage of a melee attack. We check for colliders up to the melee attack range and use Destructible components to do damage. Damage is only done if we own the Destructible.

Prepares an attack towards the given direction (degrees, unity-conform definition).

The attack may not be executed if it is on cooldown. Animation and evaluation of the attack is synced. This evaluation shall be done at the fitting time instance of the animation to allow for dodging.
