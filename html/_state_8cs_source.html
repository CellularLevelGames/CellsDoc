6 /// Extended functionality for Unity SM Behaviours and makes them complete 'States'.
7 /// Only one such state is currently allowed per State Box in the Animator Controller!
8 ///
9 /// This is intended to provide safer transitions by enforcing checks for whether a transition is
10 /// allowed for a certain state. This prevents carry-over or false trigger flags to the next states.
11 /// To do this:
12 /// * state name and allowed triggers are listed when in the SM editor
13 /// * These are checked when a state transition is requested.
14 /// * Only the currently active state can be the source for triggering a transition.
15 ///
16 /// The base classes offer for inherited behaviours:
17 /// * Awake() and Start() callbacks
18 /// * OnMyStateEnter() callback
19 /// * OnStateExit() callback
20 /// * OnStateUpdate() callback
