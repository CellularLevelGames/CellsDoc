VSCode Setup

How to set up VSCode nicely

Visual Studio Code is a lightweight Editor and IDE not dependent on specifying full projects or solutions as for the full Visual Studio. It is quite fast and highly customizable through the use of Extensions, see recommended extensions below.

Setting up VSCode for Unity

Follow this site. The steps are in short:

  1. Download and install .NetCore (use version compatible to Visual Studio 2017). Restart.
  2. Install the C# extension.
  3. Open the Unity project.
  4. Set Visual Studio Code as the default External Script Editor.
  5. Extend VSCode with further extensions, like Debugging for Unity, as listed on the website above after the setup section.
  6. Enable code completion.

Note: you may have to delete all the .sln and .csproj files in the root folder to have Unity build the configuration specialized for VSCode.

Recommended Extensions and their Settings

To install any of these, search for their name on the Extensions sidebar and click Install.

C/C++

Code highlighting and other goodies when working with C or C++. Must-have!

CSharp

Use this if working for example with Unity C# code. (Could not use the 'Hash' character in the header. The extension is actually called C# .) It also provides a formatter to work with our .cs sources. Go to the settings for this extension and check whether the formatter is enabled (Format: Enable). The formatter uses the settings in 'omnisharp.json' which is placed in the repo root folder. This way everyone shares the same settings.

Regions with Colors

Provides region highlight for C# code (see sources for examples). You can set the color with alpha channel in the settings. I use purple: rgba(132, 27, 237, 0.1)

Git Lens

This adds way more functionality, such as Diffs, History, Blame annotations etc. to your source control in VS Code. You may want to configure it to show everything under the normal Source Control sidebar, so it does not generate an additional icon to the side.

Git Graph

Allows you to show a Git Graph (or Git log) view from the Source Control side bar via a small additional icon at the top, or via the "Git Graph" button in the status bar.
It is way easier to get an overview on what is going on using this visual representation of branches etc..

Bracket Pair Colorizer 2

This will highlight cascaded brackets such as ([{}]) in your code, making your life a bit easier :)

Code Spell Checker

This is a nice spelling checker. You can disable certain file types and it will recognize CamelCase and the like as a concatenation of words. You may have to add a lot of words to your User Dictionary in the beginning though.

Hint: When having this and other kinds of checks, such as Unused-include checks (C# support), use a filter of !Photon in the bottom PROBLEMS panel to not display findings from the Photon sources we use.

PlantUML

Provides previews of plantuml code blocks in Markdown or own PlantUML files when viewing these with VS Code. Be sure to set the mode to Server. This documentation should contain some examples of how to write PlantUML diagrams in the Markdowns. Click the top right "Open Preview to the Side" button to have your Markdown rendered.

Markdown Preview Github Styling

Makes your markdown background white and is much closer to the usually rendered result this way! Also UML diagrams with black text will be readable now ;)

TODO Highlight

...lets you highlight tags you can set yourself. Typical comment flags, such as //TODO are pre-set. You may want to add the Doxygen-specific flag \todo though. You can also define your own, such as "//TONI". This is done in the settings.json, see explanations in the extension's info page.

vscode-icons

Nice icons for your folders and files, increasing explorer view usability a lot!

Vim

If you are that type of person, you can have Vim input options for VS Code with this extension. When going for Vim, I highly recommend to also use a re-mapper to have ctrl and escape set to your Caps :)

Link: Dual Key Remap on GitHub
