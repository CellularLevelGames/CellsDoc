Detailed Description

Destructibles have hit points and are affected by damage sources.

They break and their parts can be thrown by the damage source when their HP reach zero. Needs a PhotonView if syncing is active!

◆ Break()

IEnumerable<GameObject> Break ( )
inlineprivate

The Break function returns a list of GOs which represent the broken parts of the object this script is attached to.

Set a different 'broken' version of the model in the 'brokenObjectPrefab' slot. The script will find all children having a SpriteRenderer, i.e. are actually displayable GOs. All parts are expected to have suitably weighted rigidbodies attached, which may be disabled though. This method uses the 'Yield' syntax to return list elements as iterator bit by bit.

Synchronization of hitpoints via Photon.

Each value has to be serializable and is written and read to/from the stream in sequence. The data is transferred unreliable and should thus not be used to trigger any game mechanics, only visuals!
