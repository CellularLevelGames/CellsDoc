Unit Concept

There are certain similarities and differences between the different entities in a match. It makes sense, for example, to treat the player unit and army units equally regarding dealing and taking damage or status effects. These similarities need to be represented in the architecture and interface.

At the same time, we want to keep the option for the player to have different units as player unit. To also allow the player to take control of a unit during runtime, we need to be able to switch the controling part of a unit.

Unit Control Variation

The idea is to divide Unit components into common parts and those responsible for AI or Player control. Common unit parts reside on the root object, while a child 'Control' object implements the behaviour.

Common parts of each Unit:

  • SpriteRenderer with that unit's sprite
  • Collider for actual collisions
  • Animator for Sprite visuals
  • Unit Animation Controller (custom script) to control the Animator
  • Shadow Caster 2D with custom shape according to sprite
  • PhotonView and PhotonRigidbody2DView for syncing position, visuals and networking of spawning/destruction.
  • Destructible as all units are likely to have health points.

All in all, it is not possible to write all the settings to Scriptable Object assets, but we will need one prefab per unit type. For movement, these components should be on the root level of the object.

We may need to add a common "Unit" script component, to allow for easy access to Unit functionality. This script may reference the controller script and provide network sync using its PhotonView on the same object level.

Ai Units:

  • Unit controller with
  • Animator for the unit logic state machine
  • Additional colliders for the unit logics (e.g. guard range)

Player Units:

  • Needs an additional Player controller component.

Dealing and taking damage

It shall be possible to dodge melee and ranged attacks. This means and attack order does not necessarily lead to a hit. Instead an attack is initiated in a specific direction and its outcome evaluated.
