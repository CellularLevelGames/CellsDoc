Graphics

Renderer

The decision to go for 2D allows us to use a special 2D rendering pipeline. Its main benefits are the additions of realtime 2D lighting. As per our current knowledge, this removes 3D lighting possibilities at the same time, which means we can NOT mix 3D and 2D assets nicely (they would be evenly lit, no shadows)!

The 2D rendering pipeline is a part of the new Universal Rendering Pipeline (URP) concept of Unity. To set it up, a rendering pipeline asset is created and saved under Assets/Renderer/URP_2D_RenderPipeline.asset. The rendering pipeline lists which renderers are available, such as the 2D_Renderer asset we use. Some quality settings, like shadow resolution, HDR and Anti Aliasing can be set here. To switch quality from an options menu this assignment can be accessed and changed from script. Also the renderer can be selected by each camera.

The 2D_Renderer assets itself does not feature so many settings, but can for example change how transparent objects are sorted for the camera.

Post Processing

These effects are applied to the 2D version of the image, so after Rasterization has taken place (conversion from 3D to our flat 2D screen). The so-called Post Processing Profile asset (Renderer folder) holds which effects are applied with what settings.

Currently we apply bloom and vignette in our standard setting. We can tune this in the future.

Main Camera

Our main camera is available as a prefab, so we have the same settings in every scene. Most important:

  • Projection: We use perspective, as this allows us to have sprites at different z-Positions, giving a parallax effect during movement.
  • Post Processing: on
  • Environment: Used to set the background, currently only filled with a dummy grey.
  • Volume component: This defines a volume for post processing effect, where the above mentioned settings# asset is associated. The volume is set to the camera for convenience and is active globally. One could also create volumes as part of the environment and apply different effects, depending on which volume(s) the camera is currently in.

Background

The level background should be filled with some organic design. The first attempt was to create a particle system which produces cell-like floating particles in the background. There is no preset fog effect in the 2D renderer seemingly, so I used multiple transparent black layers to make particles in the back less visible. This scheme though currently is quite hardware hungry as it takes a lot of particles.
